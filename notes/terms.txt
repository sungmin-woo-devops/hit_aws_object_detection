### Phase 0


### Phase 1
    - 오토라벨링이란
        정의: 사람이 직접 그리지 않고, 알고리즘을 이용해 객체의 바운딩 박스를 자동 생성하는 과정
        목적: 대규모 데이터셋 제작 시 라벨링 시간을 단축하고 일관성을 유지가
    - 오토라벨링 종류
        1) 템플릿 기반
            - 사전에 정의된 아이콘(템플릿 이미지)을 원본 이미지와 비교해 동일/유사 객체를 탐지
            - 특징: 높은 정밀도(Precision) 확보 가능, 하지만 새로운 변형이나 다른 버전 아이콘에는 약함.
        2) 텍스트 조건 기반(VLM, Vision-Language Model)
            - "EKS 아이콘 찾아줘"처럼 텍스트 조건을 모델에 주어 탐지
            - 특징: 리콜(Recall) 향상에 유리, 새로운 아이콘 탐지 가능
        3) 하이브리드
            - 템플릿 + VLM 결과를 병합하여 정밀도와 리콜을 동시에 보완
    - ORB (Oriented FAST and Rotated BRIEF)
        - 정의: 빠르고 가벼운 특징점 검출(FAST)과 디스크립터(BRIEF)를 개선한 알고리즘
        - 특징:
            - 회전 불변(Rotation-Invariant)
            - 크기 스케일 변화에 강함
            - SIFT/SURF 대비 속도 빠름 (특허 문제 없음)
        - Phase 1에서의 역할: 템플릿 아이콘과 원본 이미지에서 동일한 특징점을 찾아 매칭
    - SIFT (Scale-Invariant Feature Transform)
        - 정의: 이미지에서 스케일/회전 변화에도 강한 로컬 특징점 검출 알고리즘
        - 특징:
            - 다양한 크기/회전에서도 특징점 유지가
            - 정밀하지만 속도가 느리고, 과거에는 특히 이슈 있었음 (현재는 만료)
        - Phase 1에서의 역할: 고정밀 매칭이 필요한 경우 ORB 대안 또는 보완재로 사용

    - RANSAC 정합 (Random Sample Consensus)
        - 정의: 이상치(Outlier)에 강한 모델 추정 알고리즘
        - 작동 방식:
            1. 랜덤 샘플로 변환 행렬(Homography 등) 추정
            2. 전체 매칭 점에서 모델과 잘 맞는 점(Inlier)만 유지
            3. 반복하여 최적 모델 선택
        - Phase 1에서의 역할: ORB/SIFT가 찾은 매칭점 중 잘못된 매칭을 제거하고, 정확한 변환 행렬 추출

    - 멀티스케일 매칭 (Multi-scale matching)
        - 정의: 원본 이미지와 템플릿을 여러 크기 비율(0.6~1.6 등)로 변환하며 탐색.
        - 목적: 아이콘이 다이어그램에서 크기 변형되어도 매칭 가능하도록 함
    
    - 회전 허용 (Rotation Invariance)
        - 정의: 템플릿 원본 이미지에서 회전된 경우에도 매칭 가능하도록 처리
        - 방법: 특징점 디스크립터에 방향(Orientation) 정보를 포함하거나, 다중 회전 템플릿 생성.

    - NMS (Non-Maximum Suppression)
        - 정의: 중복된 박스 중 가장 신뢰도가 높은 하나만 남기고 나머지 제거.
        - 파라미터: IoU=0.5 → 두 박스의 겹침 비율이 50% 이상이면 중복으로 판단.
    
    - pseudo/template.jsonl
        - 정의: 템플릿 기반 오토라벨 결과를 저장하는 JSON Lines 포맷 파일
        - 구조 예시
            ```python
            {"xyxy": [x1, y1, x2, y2], "class": "EKS", "score": 0.93, "source": "template"}

            xyxy: 좌상단(x1,y1) ~ 우하단(x2,y2) 좌표
            class: 탐지된 아이콘 클래스명
            score: 매칭 신뢰도가
            source: 탐지 방식("template" -> 템플릿 매칭)
            ```
